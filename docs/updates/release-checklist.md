---
title: 发布前检查清单
description: 最小发布检查流程，降低线上回归风险。
---

每次发布前，至少按下面清单走一遍。

## 1) 构建与基础质量

- [ ] `npm ci` 成功
- [ ] `npm run build` 成功（无 broken links / 无 MDX 编译错误）
- [ ] 若涉及首页动效：确认 `prefers-reduced-motion` 下可读性正常

## 2) 关键页面抽查

- [ ] `/`：标题、按钮、分隔线、光点动效正常
- [ ] `/home`：导航卡片与起步路径可点击、布局不乱
- [ ] `/docs`：侧边栏、目录高亮、上一页/下一页可用
- [ ] 1 个“关于”页：链接跳转正常

## 3) 响应式与主题

- [ ] 视口至少抽查 `1366x768` 与 `1920x1080`
- [ ] 浅色/深色主题都检查一轮
- [ ] 首页与文档页切换后不存在颜色串台

## 4) 发布记录

- [ ] 在 [更新日志](./changelog) 记录本次对用户有感知的改动
- [ ] 若改动较大，补截图基线（见 [截图基线规范](./screenshot-baseline)）
