把这三步走完，你就能在本站里高效自学：知道该从哪里开始、用什么工具、怎么把内容落到实践。

## 第 1 步：选一条起步路线（5 分钟）

- **只想先把站点摸清楚**：从文档总览开始浏览模块结构。  
  - 入口：[文档总览](../README.md)
- **想快速进入“能用”的状态**：按“新手必读”的顺序走一遍。  
  - 入口：[新手必读总览](./lesson/freshman.md)

## 第 2 步：把基础环境搭好（30–60 分钟）

建议按下面顺序做，避免来回踩坑：

- **PowerShell（Windows）**：基础命令与常用操作  
  - 去看：[命令行入门（PowerShell）](./basic/powershell.md)
- **VS Code**：编辑器、插件与基本工作流  
  - 去看：[VS Code 快速上手](./development/vscode.md)
- **Anaconda / Python 环境**：创建环境、包管理与常见问题  
  - 去看：[Python 环境配置（Anaconda）](./development/anaconda.md)

完成标准（自检）：

- 能打开终端、运行基础命令
- 能在 VS Code 打开一个文件夹并运行一段代码/脚本
- 能创建并激活一个独立环境

## 第 3 步：做一个最小实践（1–2 小时）

任选其一，目标是“能交付一个结果”：

- **工具实践线**：学会 Git 的最小闭环（克隆 → 修改 → 提交）  
  - 去看：[Git 入门实操](./development/git.md)
- **AI 实践线**：从一个可运行的工作流开始  
  - 去看：[Dify 入门](./ai-tools/dify.md) 或 [ComfyUI 入门](./ai-tools/comfyui.md)

完成标准（自检）：

- 你能复述“我现在在做什么、下一步做什么”
- 你能把结果记录下来（笔记/截图/仓库提交均可）
