# 首页视口回归矩阵（P0）

用于每次首页布局/动效改动后的最小回归检查。

## 测试环境

- 浏览器：Chrome（最新稳定版）
- 缩放：100%
- 系统字体缩放：100%
- 主题：浅色/深色各测一轮

## 视口矩阵

- 1366 x 768
- 1536 x 864
- 1728 x 1117（常见半屏）
- 1920 x 1080
- 2560 x 1440
- 3440 x 1440（带鱼屏）

## 验收项（每个视口都要过）

1. 首页标题与副文案无重叠、无溢出、无明显裁切。
2. 分隔线方向正确（单栏为横向，双栏为纵向），不出现“粗块/断裂”。
3. 按钮保持竖向排列，可正常点击，hover 不错位。
4. 光点与拖尾运动连续，无明显瞬移或卡顿。
5. 导航栏与页脚在首页样式下可读性正常。
6. 在 `prefers-reduced-motion: reduce` 下，页面仍完整可读。

## 截图基线建议

当首页或导航页有较大 UI 调整时，建议补充最小截图基线，便于回归对比。

### 建议页面

- `/`（落地页）
- `/home`（站内导航）
- 任意文档页（包含目录 + 翻页卡片）

### 建议视口

- 1366 x 768
- 1920 x 1080
- 2560 x 1440

### 建议命名

- `YYYYMMDD_home_1366x768_light.png`
- `YYYYMMDD_home_1366x768_dark.png`

截图可放在仓库任意约定位置（例如 `docs/about/assets/`），只要团队统一即可。  
更完整的命名与目录规则见：[截图基线规范](./screenshot-baseline)。
