description: 图片命名、格式、尺寸与引用方式的统一约定（避免部署 404 与仓库膨胀）。
---

本规范适用于站内文档与静态资源（Docusaurus）。

## 1) 存放位置

- 站点静态资源统一放在：`static/img/`
- 文档示例图建议放在：`static/img/examples/`

## 2) 命名规则（强制）

为避免 Windows 本地正常、GitHub Pages（Linux）上 404 的问题：

- **全小写**
- 用 **`-`** 连接单词
- 避免空格与中文

推荐示例：

- `vscode-logo.webp`
- `docker-desktop.webp`
- `tjufe-emblem-384.webp`

## 3) 格式与体积（强烈建议）

- 优先使用 **WebP**（体积更小）
- 仅在必要时使用 PNG（需要透明/无损）
- 单张图片尽量控制在 **200KB** 内（示例图一般不需要原图分辨率）

建议的导出宽度（经验值）：

- 文档截图：最长边 **1200px** 以内
- Logo/图标：**64–384px**（按用途）

## 4) 引用方式（统一）

文档中统一使用绝对路径引用（最直观、最少坑）：

```md
![描述性文案](/img/examples/your-image.webp)
```

避免混用：

- MDX 的 `require(...)`
- HTML `<img ...>`

（除非确实需要控制宽度/布局；需要时也优先用 Markdown + CSS 方案。）

## 5) alt 文案（可读性与无障碍）

不要用文件名当 alt。建议写“这张图在说明什么”：

- ✅ `![VS Code 下载页面示例](...)`
- ❌ `![vscode](...)`
